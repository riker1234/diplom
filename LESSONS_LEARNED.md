# Чему мы научились за время разработки

## Парсинг: Selenium, Requests, httpx

В ходе разработки мы работали с несколькими инструментами для сбора данных. Для Ozon и Wildberries использовали **Selenium** — библиотеку для управления браузером. Научились запускать браузер в фоновом режиме, ждать загрузки страниц и извлекать нужные данные из JavaScript-приложений. До этого проекта не было понимания почему обычный HTTP-запрос не работает на современных сайтах — теперь есть.

Для Ситилинка хватило **httpx** — более быстрой альтернативы стандартному requests. Научились делать асинхронные запросы, обрабатывать редиректы и извлекать данные из HTML через регулярные выражения.

Также познакомились с тем как устроены CDN-серверы на примере Wildberries — изображения хранятся на десятках серверов по предсказуемым адресам, и научились генерировать эти адреса программно.

---

## База данных: PostgreSQL, SQLAlchemy

Работали с **PostgreSQL** как основной СУБД и **SQLAlchemy** как ORM-прослойкой между Python и базой. Научились описывать таблицы через Python-классы, писать фильтры, использовать OR/AND условия и правильно обрабатывать NULL-значения в запросах — это оказалось нетривиально, потому что `NULL != NULL` в SQL работает не так как ожидаешь.

Разобрались с миграциями через **Alembic** — как версионировать схему базы и применять изменения без потери данных.

Научились думать о проектировании схемы исходя из реальных запросов, а не абстрактных принципов. Поняли когда денормализация оправдана, а когда она только усложняет жизнь. В нашем случае шесть отдельных таблиц под каждую категорию товаров оказались удобнее одной общей.

---

## Бэкенд: FastAPI, Python

Основной фреймворк — **FastAPI**. Научились строить REST API: описывать эндпоинты, валидировать входные данные через **Pydantic**-схемы, настраивать CORS для работы с фронтендом. Познакомились с концепцией dependency injection — как FastAPI передаёт сессию базы данных в каждый обработчик через `Depends`.

Написали рекомендательную систему на чистом Python без сторонних ML-библиотек. Научились строить логику на основе фильтров и скоринга — оказалось что для практических задач это работает не хуже сложных алгоритмов, а понять и отлаживать намного проще.

Столкнулись с тем что бизнес-логика требует аккуратного тестирования — несколько багов были незаметны на первый взгляд: функции работали без ошибок, но возвращали неправильные результаты.

---

## Фронтенд: React, TypeScript, Tailwind CSS, Vite

Фронтенд написан на **React** с **TypeScript**. Научились работать с хуками — `useState` для локального состояния, `useEffect` для загрузки данных при изменении параметров, `useNavigate` и `useLocation` для навигации между страницами. Разобрались как правильно передавать данные между страницами через `state` при навигации.

С **TypeScript** пришлось разобраться в важной детали: интерфейсы и типы существуют только на этапе написания кода и полностью стираются при сборке. Попытка импортировать интерфейс как обычное значение приводит к ошибке — нужен специальный синтаксис `import type`.

**Tailwind CSS** научил подходу utility-first — стилизация прямо в разметке без отдельных CSS-файлов. Сначала кажется непривычно, но быстро становится удобным для построения компонентов.

**Vite** использовали как инструмент сборки. Узнали как настраивать dev-сервер, в частности что по умолчанию он слушает только localhost и недоступен с других устройств — нужна явная настройка `host: '0.0.0.0'`.
