# Чему мы научились за время разработки

## Парсинг: Selenium, Requests, httpx

В ходе разработки  работал с несколькими инструментами для сбора данных. Для Ozon и Wildberries использовал **Selenium** — библиотеку для управления браузером + веб двайвер для запуска в фоне. Научился работать с данной библиотекой.

Для Ситилинка хватило **httpx** — более быстрой альтернативы стандартному requests. Научился делать асинхронные запросы, обрабатывать редиректы и извлекать данные из HTML через регулярные выражения.

Также познакомился с тем как устроены CDN-серверы на примере Wildberries — изображения хранятся на десятках серверов по предсказуемым адресам, и научились генерировать эти адреса программно.

---

## База данных: PostgreSQL, SQLAlchemy

Работал с **PostgreSQL** как основной СУБД и **SQLAlchemy** как ORM-прослойкой между Python и базой. Научился описывать таблицы через Python-классы, писать фильтры, использовать OR/AND условия.

  Работал с миграциями через **Alembic** — как версионировать схему базы и применять изменения без потери данных.

---

## Бэкенд: FastAPI, Python

Основной фреймворк — **FastAPI**. Продожаем учиться строить REST API: описывать эндпоинты, валидировать входные данные через **Pydantic**-схемы, настраивать CORS для работы с фронтендом. Познакомились с концепцией dependency injection — как FastAPI передаёт сессию базы данных в каждый обработчик через `Depends`.

Написал рекомендательную систему на чистом Python без сторонних ML-библиотек.


---

## Фронтенд: React, TypeScript, Tailwind CSS, Vite

Фронтенд написан на **React** с **TypeScript**. Продолжаем изучать данную библиотеку и работать с ней

С **TypeScript** пришлось разобраться в важной детали: интерфейсы и типы существуют только на этапе написания кода и полностью стираются при сборке. Попытка импортировать интерфейс как обычное значение приводит к ошибке — нужен специальный синтаксис `import type`.

**Tailwind CSS** научился подходу utility-first — стилизация прямо в разметке без отдельных CSS-файлов. Сначала кажется непривычно, но быстро становится удобным для построения компонентов.

**Vite** использовал как инструмент сборки. Узнали как настраивать dev-сервер, в частности что по умолчанию он слушает только localhost и недоступен с других устройств — нужна явная настройка `host: '0.0.0.0'`.
